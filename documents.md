# はじめに
Vuetifyとは、Vueで広く使われているUIフレームワークライブラリです。

ある程度綺麗な画面を作れる上、付属のコンポーネントにも便利なものが多くてとても良いライブラリなのですが、その分柔軟性も高く、油断すると画面の統一感が無くなってしまうことがあります。

そんな時のため、指標とできるようなガイドラインを作成しました。
よろしければぜひご活用ください。


# ガイドライン
## 色
Vuetifyでは以下のように、役割に応じて色を設定することができます。
```
        colors: {
          background: '#FFFFFF',
          surface: '#FFFFFF',
          'surface-bright': '#FFFFFF',
          'surface-light': '#EEEEEE',
          'surface-variant': '#424242',
          'on-surface-variant': '#EEEEEE',
          primary: '#1867C0',
          'primary-darken-1': '#1F5592',
          secondary: '#48A9A6',
          'secondary-darken-1': '#018786',
          error: '#B00020',
          info: '#2196F3',
          success: '#4CAF50',
          warning: '#FB8C00'
        },
```

無闇に色を使ってしまう画面が見づらくなってしまうため、**基本的に役割が設定された色のみ使う**ようにしましょう。

色の設定はcreateVuetifyのthemeというオプションで追加編集が可能です。必要に応じて編集してください。


## レイアウト
レイアウトは統一感を出すために非常に重要な要素です。
画面内で何をどのように配置するか、必ず事前にチームで決めておくべきでしょう。

以下のような、ほとんどのアプリケーションに存在する要素について、予めどのように配置していくかを考えておきましょう。
- グローバルヘッダー
- サイドバー
- 画面ヘッダー
- 画面コンテンツ
- フッター

幸い、Vuetifyにはこれらの大きなレイアウトのためのコンポーネントがすでに用意されているため、活用していきましょう。
参考: https://vuetifyjs.com/ja/features/application-layout/


## 余白
余白はレイアウトと混同しがちな概念ですが、本記事ではレイアウトは画面構成全体のことで、余白は要素同士の間隔のことを示すようにします。

余白も規則を決めておかないと、まとまりが無くなる上にコンテンツが見難くなります。
「何と何はどれくらい余白を取るべきか」を明文化するようにしましょう。
例として以下です。
- コンテンツの横幅は〇〇px
- タイトルとコンテンツの間は〇〇pxさま
- リストの間は〇〇px
...etc

余白関連のスタイルはVuetifyの方でユーティリティクラスとして定義されてますので、こちらも積極的に使うようにしましょう。
参考: https://vuetifyjs.com/ja/styles/spacing/#section-4f7f304465b9


## コンポーネント
### props
Vuetifyのコンポーネントはpropsによって見た目が柔軟に変化するようになっています。
ということは、しっかりとルールを決めていないと、画面が散らかってしまうということでもあります。
特に気をつけた方が良いpropsについて記載します。

#### variants
主に見た目のスタイルを決めるpropsです。'outlined'、'plain'などスタイル名の文字列で指定します。
スタイルごとに役割を決めた方が良いでしょう。

#### density
高さを指定するpropsです。基本的に'default'、'compact'があります。
画面の情報密度に応じて使い分けることで、統一感のあるレイアウトを実現できます。

#### hide-details
入力フォームでエラー文やヒントなどのスペースを削除するために使用するpropsです。
有効にしたままにすると結構邪魔になることがあるため、適切な場面での使用を心がけましょう。

### アクション要素

#### ボタン (v-btn)
ボタンはWEB画面において、操作の起点となる非常に重要な要素です。
色も考慮し、テキストやアイコンもどのように出すかもしっかり決めましょう。

このガイドでは以下のように定めます。
| ボタンの種類      | 用途例         | 使用数         | 色                | variant           | アイコン                        | 備考         |
|------------------|--------------------|----------------|-------------------|-------------------|------------------------------|--------------|
| プライマリ       | 確定ボタンなど、画面内で最も重要な操作に対して使用する         | 画面に1つ       | primary           | contained         | prepend-icon: mdi-check       |
| セカンダリ       | キャンセルボタンやダウンロードボタンなど、補助的な操作に対して使用する       | 複数可          | secondary         | outlined          | なし                          |
| テキスト         | リンクなど優先度が低い操作に対して使用する             | 複数可          | default           | text              | なし                          |
| アイコンボタン   | ファイルのダウンロードやslackのリンクなど、操作内容が明確な場合のみ使用可 | 複数可          | primary/secondary | icon              | 必須（操作内容が明確な場合）   |v-fabの使用と、ツールチップでホバー時に詳細の表示が必須

- 高さは基本的にdefaultを使用する。

#### チップ (v-chip)
チップは主にタグやステータス、選択された項目を表示するために使用します。
情報の分類や状態表示において重要な役割を果たします。

このガイドでは以下のように定めます。
| チップの種類 | 用途例 | 色 | variant | アイコン | 備考 |
|-------------|--------|-------|---------|----------|------|
| ステータス | 完了、進行中、エラーなどの状態表示 | success/warning/error | elevated | prepend-icon: 状態に応じて | 削除不可 |
| タグ | カテゴリ分類、ラベル表示 | primary/secondary | outlined | なし | 削除可能な場合はclosableを使用 |
| 選択項目 | フィルター、選択済みアイテム | primary | elevated | なし | 削除可能、クリック可能 |
| 情報表示 | 数値、件数などの補足情報 | default | text | なし | 削除不可、クリック不可 |

- 同一画面内では統一したvariantを使用する
- 削除可能なチップには必ずclosableプロパティを設定する


### フォーム・入力

#### テキスト入力 (v-text-field)
テキスト入力は最も基本的なフォーム要素です。
ユーザビリティとバリデーション表示の一貫性が重要です。

このガイドでは以下のように定めます。
| 項目 | 設定値 | 備考 |
|------|--------|------|
| variant | outlined | 統一感のため全てoutlinedを使用 |
| density | default | 基本的にdefault、省スペース時のみcompact |
| hide-details | false | エラーメッセージ表示のため基本的に無効 |
| clearable | true | 入力内容をクリアできるよう設定 |
| placeholder | 具体的な入力例 | "例: 山田太郎" のように具体例を記載 |

- 必須項目には`required`とアスタリスク(*)を表示
- バリデーションエラーは`error-messages`で表示
- ラベルは簡潔で分かりやすく記載

#### テキストエリア (v-textarea)
複数行のテキスト入力に使用します。
基本的にはテキスト入力と同じルールを適用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| variant | outlined | テキスト入力と統一 |
| rows | 3-5 | 用途に応じて適切な行数を設定 |
| auto-grow | true | 内容に応じて高さを自動調整 |
| counter | 文字数制限時に設定 | 最大文字数を表示 |

#### セレクト (v-select)
選択肢から単一または複数の項目を選択するために使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| variant | outlined | フォーム要素と統一 |
| density | default | 基本的にdefault使用 |
| clearable | true | 選択解除を可能にする |
| chips | multiple時にtrue | 複数選択時は選択項目をチップで表示 |
| no-data-text | "データがありません" | 選択肢がない場合のメッセージ |

- 選択肢は5個以下の場合はラジオボタンの使用を検討
- 多数の選択肢がある場合はオートコンプリートの使用を検討

#### オートコンプリート (v-autocomplete)
多数の選択肢から検索しながら選択する場合に使用します。
セレクトの代替として、選択肢が多い場合に有効です。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| variant | outlined | フォーム要素と統一 |
| clearable | true | 選択解除を可能にする |
| no-filter | false | 検索機能を有効にする |
| chips | multiple時にtrue | 複数選択時は選択項目をチップで表示 |
| no-data-text | "該当する項目がありません" | 検索結果がない場合のメッセージ |

- 選択肢が10個以上の場合に使用を検討
- 検索しやすいよう適切なitem-titleを設定

#### チェックボックス (v-checkbox)
複数選択が可能な項目に使用します。
独立した選択項目として機能します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | primary | 選択時の色をプライマリカラーに統一 |
| density | default | 基本的にdefault使用 |
| hide-details | true | 単体使用時はエラー表示領域を非表示 |

- 利用規約同意など、必須チェック項目には適切なバリデーションを設定
- グループ使用時は見出しを明確に表示

#### ラジオボタン (v-radio)
単一選択の項目に使用します。
選択肢が5個以下の場合に推奨します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | primary | 選択時の色をプライマリカラーに統一 |
| density | default | 基本的にdefault使用 |

- v-radio-groupで囲んでグループ化する
- 選択肢は縦並びを基本とし、横幅に余裕がある場合のみ横並び

#### スイッチ (v-switch)
ON/OFFの切り替えに使用します。
設定項目や機能の有効/無効に適しています。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | primary | 有効時の色をプライマリカラーに統一 |
| inset | false | 基本的には標準スタイルを使用 |
| hide-details | true | 単体使用時はエラー表示領域を非表示 |

- ラベルは「〇〇を有効にする」など、動作が明確になるよう記載
- 即座に反映される設定に使用（保存ボタン不要）

#### スライダー (v-slider)
数値の範囲選択に使用します。
直感的な値設定が可能です。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | primary | スライダーの色をプライマリカラーに統一 |
| step | 用途に応じて設定 | 適切な刻み幅を設定 |
| show-ticks | 必要に応じてtrue | 目盛り表示で値を分かりやすく |
| thumb-label | true | 選択中の値を表示 |

- 最小値・最大値・現在値を明確に表示
- 精密な値設定が必要な場合はテキスト入力との併用を検討

#### ファイル入力 (v-file-input)
ファイルのアップロードに使用します。
セキュリティとユーザビリティの両立が重要です。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| variant | outlined | フォーム要素と統一 |
| show-size | true | ファイルサイズを表示 |
| clearable | true | 選択解除を可能にする |
| accept | 適切な形式を指定 | 受け入れ可能なファイル形式を制限 |

- 受け入れ可能なファイル形式・サイズを明記
- プレビュー機能の実装を推奨（画像ファイルの場合）

### レイアウト・コンテナ

#### コンテナ (v-container)
ページ全体のコンテンツ幅を制御し、レスポンシブレイアウトの基盤となります。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| fluid | false | 基本的には最大幅制限ありで使用 |
| class | 用途に応じて設定 | pa-4, pa-6などでパディングを統一 |

- 画面全体に1つのコンテナを配置することを基本とする
- フルワイド表示が必要な場合のみfluidを使用

#### グリッドシステム (v-row, v-col)
レスポンシブなカラムレイアウトを実現します。
12カラムシステムを基本とします。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| no-gutters | false | 基本的にはガッター（余白）ありで使用 |
| cols | 12, 6, 4, 3を基本 | 均等分割を心がける |
| sm, md, lg, xl | ブレイクポイント対応 | レスポンシブ対応を必須とする |

- モバイルファーストでcolsを設定
- 複雑なレイアウトは避け、シンプルな分割を心がける

#### カード (v-card)
関連するコンテンツをグループ化して表示します。
最も多用されるコンテナコンポーネントです。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| variant | elevated | 基本的にはelevatedで影をつける |
| class | pa-4, pa-6 | 内部パディングを統一 |

- カード内の構成要素は v-card-title, v-card-text, v-card-actions に分ける
- アクションボタンは右寄せで配置

#### ディバイダー (v-divider)
コンテンツの区切りを明確にします。
視覚的な分離に使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| thickness | 1 | 基本的には1pxの細い線 |
| opacity | 0.12 | 控えめな表示で統一 |

- リスト項目間の区切りに使用
- セクション間の区切りにも活用

#### エクスパンションパネル (v-expansion-panel)
折りたたみ可能なコンテンツ表示に使用します。
FAQや設定項目の表示に適しています。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| variant | default | 基本スタイルを使用 |
| multiple | false | 基本的には単一展開 |

- タイトルは分かりやすく簡潔に記載
- 展開時のコンテンツは適切にパディングを設定

### ナビゲーション

#### アプリバー (v-app-bar)
アプリケーションの最上部に配置するメインナビゲーションです。
ブランディングと主要機能への導線を提供します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| app | true | アプリケーション全体のレイアウトに影響 |
| color | primary | ブランドカラーで統一 |
| density | default | 基本的にはdefault使用 |
| flat | false | 影をつけて立体感を表現 |

- ロゴ・タイトルは左端に配置
- ユーザー関連機能は右端に配置
- モバイル時はハンバーガーメニューを表示

#### ナビゲーションドロワー (v-navigation-drawer)
サイドメニューとして使用する折りたたみ可能なナビゲーションです。
アプリバーと連携して階層的なナビゲーションを実現します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| app | true | アプリケーション全体のレイアウトに影響 |
| temporary | モバイル時true | デスクトップでは常時表示、モバイルでは一時表示 |
| width | 280 | 適切な幅で統一 |

- メニュー項目はv-listで構造化
- 現在ページはactive状態で明示
- グループ化して階層を分かりやすく表示

#### ボトムナビゲーション (v-bottom-navigation)
モバイルアプリでの主要機能へのアクセスに使用します。
3-5個の主要機能に限定して配置します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| height | 60 | タップしやすい高さに設定 |
| grow | false | アイコンとラベルのバランスを保持 |

- アイコンとラベルはセットで表示
- 現在のページを明確にハイライト
- 5個以下のメインメニューに限定

#### タブ (v-tabs)
同一レベルのコンテンツ間を切り替える際に使用します。
情報の整理と表示切り替えに効果的です。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | primary | アクティブタブの色を統一 |
| density | default | 基本的にはdefault使用 |
| show-arrows | モバイル時true | 横スクロール時の操作性向上 |

- タブ数は7個以下に制限
- タブ名は簡潔で分かりやすく記載
- アクティブタブを明確に表示

#### ブレッドクラム (v-breadcrumbs)
現在位置の階層を示すナビゲーション補助要素です。
深い階層のページで現在位置を明確にします。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| divider | "/" | 区切り文字を統一 |
| items | 階層に応じて設定 | 適切なリンクとラベルを設定 |

- 第2階層以降で表示を開始
- 最大5階層まで表示
- 現在ページはリンクなしで表示

#### ページネーション (v-pagination)
長いリストやテーブルの分割表示に使用します。
パフォーマンスとユーザビリティの向上に寄与します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | primary | ページ番号の色を統一 |
| length | データに応じて設定 | 適切なページ数を算出 |
| show-first-last-page | true | 最初・最後のページへのジャンプを提供 |

- 1ページあたりの表示件数を明記
- 現在ページ番号を明確に表示
- 前後ページへの移動を容易にする

### データ表示

#### データテーブル (v-data-table)
大量のデータを表形式で表示・操作するコンポーネントです。
ソート、フィルタリング、ページネーションなどの機能を提供します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| density | default | 基本的にはdefault、省スペース時のみcompact |
| show-select | 必要時のみtrue | 複数選択が必要な場合のみ有効 |
| hover | true | 行のホバー効果で視認性向上 |
| loading | データ取得中true | ローディング状態を明示 |

- ヘッダーには適切なソート機能を実装
- 1ページあたりの表示件数は25-50件を基本とする
- アクションボタンは最右列に配置

#### リスト (v-list)
構造化されたデータを縦方向に表示するコンポーネントです。
ナビゲーションやオプション選択に広く使用されます。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| density | default | 基本的にはdefault使用 |
| nav | ナビゲーション時true | ナビゲーション用途の場合に設定 |
| lines | 内容に応じて設定 | 'one', 'two', 'three'を適切に選択 |

- アイコンは左端に統一配置
- サブタイトルは補足情報に使用
- アクションは右端に配置

### フィードバック・通知

#### アラート (v-alert)
重要な情報をユーザーに伝えるためのコンポーネントです。
成功、警告、エラーなどの状態に応じて適切に使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| type | success/info/warning/error | 内容に応じて適切なタイプを選択 |
| closable | 一時的なメッセージはtrue | ユーザーが閉じられるよう設定 |
| prominent | 重要度高い場合true | 強調表示が必要な場合のみ |
| variant | tonal | 基本的にはtonalで統一 |

- エラー時は具体的な解決方法を併記
- 成功メッセージは自動で消える設定も検討
- アイコンは自動で設定されるため手動設定不要

#### バナー (v-banner)
ページ上部に表示する重要な通知やアナウンスに使用します。
アラートよりも目立つ表示が必要な場合に使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | 内容に応じて設定 | 重要度に応じた色を使用 |
| sticky | 重要な場合true | スクロール時も表示を維持 |

- システムメンテナンス告知などに使用
- アクションボタンは1-2個まで
- 長期間表示する重要な情報に限定

#### スナックバー (v-snackbar)
一時的なフィードバックメッセージを画面下部に表示します。
操作の結果を簡潔に伝える際に使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| timeout | 4000-6000 | 4-6秒で自動非表示 |
| location | 'bottom' | 基本的には画面下部 |
| color | 内容に応じて設定 | success/error/infoを使い分け |

- メッセージは簡潔に（1-2行以内）
- アクションボタンは1つまで
- 複数同時表示は避ける

#### プログレス (v-progress-circular, v-progress-linear)
処理の進行状況を視覚的に表示するコンポーネントです。
ユーザーの待機時間を明確にします。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | primary | 基本的にはプライマリカラー |
| indeterminate | 進行度不明時true | 処理時間が不明な場合 |
| size | 用途に応じて設定 | 画面レイアウトに応じて調整 |

- 3秒以上かかる処理では必ず表示
- 進行度が分かる場合は具体的な%を表示
- 円形は小さな領域、線形は全体の処理に使用

#### スケルトンローダー (v-skeleton-loader)
コンテンツの読み込み中に表示する仮のレイアウトです。
実際のコンテンツ構造を模した表示で待機時間を短く感じさせます。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| type | コンテンツに応じて設定 | 'card', 'list-item'など |
| loading | データ取得中true | データ取得状態と連動 |

- 実際のコンテンツ構造に近い形状で作成
- 2秒以上のデータ取得時に使用
- アニメーション効果で動的な印象を演出

#### バッジ (v-badge)
数値や状態を小さく表示するコンポーネントです。
通知数や状態表示に使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | 内容に応じて設定 | error（緊急）、warning（注意）など |
| dot | 数値不要時true | 単純な状態表示の場合 |
| max | 99 | 大きな数値は'99+'で省略表示 |

- 未読数やエラー数の表示に使用
- 0の場合は非表示にする
- 緊急度に応じて色を使い分け

### オーバーレイ

#### ダイアログ (v-dialog)
モーダルウィンドウとして確認や詳細入力画面を表示します。
ユーザーの注意を集中させる重要な操作に使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| max-width | 600px | 基本的な最大幅を設定 |
| persistent | 重要な場合true | 背景クリックで閉じないよう設定 |
| scrollable | 内容が長い場合true | コンテンツのスクロールを可能にする |

- タイトル、コンテンツ、アクションの3つのセクションに分ける
- アクションボタンは右寄せで「キャンセル」「確定」の順で配置
- 破壊的操作（削除など）の確認に必ず使用

#### ボトムシート (v-bottom-sheet)
モバイル向けの画面下部から表示されるモーダルです。
モバイルでの操作性を重視する場合に使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| inset | false | 基本的にはフルワイド表示 |
| persistent | false | 基本的には背景クリックで閉じる |

- モバイル専用の操作として使用
- アクション選択（共有、削除など）に適している
- デスクトップではダイアログを使用

#### メニュー (v-menu)
ボタンやアイコンに関連する追加操作を表示します。
コンテキストメニューやドロップダウンメニューとして使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| close-on-click | true | 項目選択時に自動で閉じる |
| offset-y | true | アクティベーター要素と重ならないよう配置 |

- メニュー項目は7個以下に制限
- アイコンと文字を組み合わせて分かりやすく表示
- 危険な操作（削除など）は色で区別

#### ツールチップ (v-tooltip)
要素にホバーした際の補足説明を表示します。
アイコンボタンや省略されたテキストの説明に使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| location | 'top' | 基本的には上部に表示 |
| open-delay | 500 | 0.5秒後に表示開始 |

- 説明文は1-2行以内で簡潔に記載
- アイコンボタンには必ず設定
- 操作方法や機能説明に使用

#### オーバーレイ (v-overlay)
画面全体を覆うオーバーレイ表示を作成します。
カスタムモーダルやローディング画面の基盤として使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| opacity | 0.46 | 適度な透明度で背景を暗くする |
| persistent | 用途に応じて設定 | クリックで閉じるかどうかを制御 |

- 全画面ローディング時に使用
- カスタムモーダルの背景として使用
- 単体使用は避け、他のコンポーネントと組み合わせ

### ピッカー・選択

#### 日付ピッカー (v-date-picker)
カレンダーから日付を選択するコンポーネントです。
予約システムやスケジュール管理で使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | primary | 選択日の色をプライマリカラーに統一 |
| locale | 'ja' | 日本語表示に設定 |
| first-day-of-week | 0 | 日曜日を週の始まりに設定 |

- 入力フォームとセットで使用することを基本とする
- 過去日制限や未来日制限を適切に設定
- 範囲選択が必要な場合は専用の実装を検討

#### 時間ピッカー (v-time-picker)
時刻を選択するコンポーネントです。
予約システムや時刻設定で使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | primary | 選択時刻の色をプライマリカラーに統一 |
| format | '24hr' | 24時間表示を基本とする |

- 日付ピッカーと組み合わせて使用することが多い
- 営業時間や利用可能時間の制限を適切に設定
- 15分刻みなど適切な時間間隔を設定

#### 色ピッカー (v-color-picker)
色を選択するコンポーネントです。
テーマ設定やカスタマイズ機能で使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| mode | 'hexa' | 16進数表示を基本とする |
| show-swatches | true | プリセットカラーを表示 |

- デザインカスタマイズ機能に使用
- プリセットカラーにブランドカラーを含める
- 一般ユーザー向けにはシンプルなカラーパレットを検討

#### 評価 (v-rating)
星やハートでの評価表示・入力に使用します。
レビューシステムや満足度調査で使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | 'amber' | 星の色は黄色系で統一 |
| half-increments | 用途に応じて設定 | 0.5刻みの評価が必要な場合 |
| readonly | 表示のみの場合true | 評価表示と入力で使い分け |

- 5段階評価を基本とする
- 評価の説明（1=悪い、5=良い）を併記
- 未評価状態と評価済み状態を明確に区別

### メディア・表示

#### アバター (v-avatar)
ユーザーの顔写真や名前の頭文字を円形で表示します。
ユーザー識別や署名表示に使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| size | 40px | 基本サイズを40pxに統一 |
| color | 画像なしの場合設定 | ランダム色またはユーザー固有色 |

- 画像がない場合は名前の頭文字を表示
- ユーザーリストでは統一サイズを使用
- ヘッダーやプロフィールでは大きめサイズを使用

#### アイコン (v-icon)
Material Design Iconsを使用してアクションや状態を表現します。
UIの視認性向上と国際化対応に重要です。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| color | 親要素に合わせる | 基本的には自動継承 |
| size | 'default' | コンテキストに応じて調整 |

- Material Design Iconsを基本とする
- アイコンの意味は国際的に通用するものを選択
- 装飾目的での使用は控えめにする
- アイコンボタンには必ずツールチップを設定

#### 画像 (v-img)
最適化された画像表示コンポーネントです。
レスポンシブ対応とローディング処理を内蔵しています。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| aspect-ratio | 用途に応じて設定 | 16:9, 4:3, 1:1など適切な比率 |
| cover | true | 基本的にはcover表示 |
| lazy-src | 必要に応じて設定 | ローディング用の軽量画像 |

- alt属性は必ず設定する
- 適切なアスペクト比を維持
- ローディング状態の表示を考慮
- レスポンシブ対応のためsrcsetを活用

#### カルーセル (v-carousel)
複数の画像やコンテンツをスライド表示します。
ギャラリーやプロモーション表示に使用します。

| 項目 | 設定値 | 備考 |
|------|--------|------|
| height | 適切な高さを設定 | レスポンシブ対応を考慮 |
| show-arrows | true | ナビゲーション矢印を表示 |
| cycle | false | 基本的には手動操作のみ |

- スライド数は7枚以下に制限
- 自動再生は避け、ユーザー操作を基本とする
- インジケーターで現在位置を明示
- モバイルではスワイプ操作に対応
